## 1. プロジェクト概要

**プロジェクト名**: FreshKeeper（フレッシュキーパー）

**サービス概要**: 食品の在庫と賞味期限・消費期限を管理し、期限が近づいたらお知らせをする、保持している食品を一覧で管理するWebアプリケーション

**バージョン**: 1.0（初期版）

**作成日**: 2026-01-24

---

## 2. 背景と目的

### 2.1 背景・きっかけ
- 買った食品の賞味期限を切らしてしまうことが多い
- 同じ食品を重複して買ってしまうことがある
- 一人暮らしを始めたばかりの人が、食品管理に困っている

### 2.2 解決したい課題
1. 食品の賞味期限切れによる食品ロスの削減
2. 重複購入の防止
3. 冷蔵庫・食品棚の在庫の可視化
4. 賞味期限が近づいた食品の早期発見と通知

### 2.3 プロジェクトの目的
一人暮らしを始めたばかりの人が、簡単に食品を管理でき、賞味期限切れや重複購入を防ぐことで、食品ロスを削減し、経済的負担を軽減する。

---

## 3. ターゲットユーザー

### 3.1 主要ターゲット
- **一人暮らしを始めたばかりの人**
  - 学生（大学生、専門学校生）
  - 新社会人
  - 年齢層: 18〜25歳

### 3.2 ユーザーの特徴
- 食品管理の経験が少ない
- 忙しくて細かい管理が難しい
- スマートフォンを日常的に使用している

---

## 4. 差別化ポイント

### 4.1 類似サービス
- **Limiter**: 既存の食品賞味期限管理アプリ

### 4.2 FreshKeeperの差別化ポイント
1. **シンプルで使いやすいUI**
   - 一人暮らしを始めたばかりの人でも直感的に使える

2. **数量管理機能**
   - 食品ごとに数量を管理し、一部消費した場合も追跡可能

3. **消費期限と賞味期限の区別**
   - 消費期限は厳密に、賞味期限は余裕ありとして通知を調整

4. **音声入力による食品登録**（Phase 2で実装予定）
   - 冷蔵庫から食品を取り出したときに、音声で食品名と賞味期限を登録できる
   - 手が塞がっていても、ハンズフリーで登録可能

## 5. 主要機能（MVP: Minimum Viable Product）

### 5.1 必須機能

#### 5.1.1 ユーザー管理
- [ ] ユーザー登録（メール/パスワード）
- [ ] Googleログイン
- [ ] ログイン/ログアウト
- [ ] パスワードリセット

#### 5.1.2 食品登録・管理
- [ ] 食品の手動登録（テキスト入力）
  - 食品名
  - 数量・単位（例: 1本、6個、500ml）
  - 購入日（任意）
  - 期限種別（消費期限 / 賞味期限）
  - 期限日
  - カテゴリ（冷蔵、冷凍、常温など）
  - 保存場所（冷蔵庫、冷凍庫、食品棚など）
- [ ] 食品情報の編集
- [ ] 数量の部分消費（例: 6個中2個消費 → 残り4個）
- [ ] 食品の削除（消費完了時）
- [ ] 食品一覧表示
  - 期限順にソート
  - カテゴリ別フィルター
  - 保存場所別フィルター

#### 5.1.3 通知機能
- [ ] 期限が近づいた食品の通知
  - 消費期限: 3日前、1日前、当日に通知
  - 賞味期限: 7日前、3日前、当日に通知
  - 通知設定のカスタマイズ可能
- [ ] Webプッシュ通知
- [ ] メール通知（オプション）

#### 5.1.4 検索機能
- [ ] 食品名で検索
- [ ] 保持している食品を一覧で確認
  - 「牛乳を既に持っているか」をすぐに確認できる

## 6. ユーザーストーリー

### 6.1 食品登録
```
As a ユーザー
I want 買い物から帰ってきたときに、食品を素早く登録したい
So that 簡単に食品を記録できる
```

### 6.2 在庫確認
```
As a ユーザー
I want スーパーで買い物中に、冷蔵庫に牛乳があるか確認したい
So that 重複購入を防ぐことができる
```

### 6.3 賞味期限通知
```
As a ユーザー
I want 賞味期限が近い食品を通知してほしい
So that 食品を無駄にせず、計画的に消費できる
```

### 6.4 食品一覧
```
As a ユーザー
I want 冷蔵庫にある食品を一覧で見たい
So that 何があるか把握して、献立を考えられる
```

### 6.5 数量管理
```
As a ユーザー
I want 卵を6個中2個使ったことを記録したい
So that 残りの数量を正確に把握できる
```

## 7. 機能要件詳細

### 7.1 食品登録機能
- **登録項目**:
  | 項目 | 必須 | 説明 |
  |-----|-----|------|
  | 食品名 | ○ | 自由入力 |
  | 数量 | ○ | 数値入力（デフォルト: 1） |
  | 単位 | ○ | 選択式（個、本、パック、g、ml、袋など） |
  | 期限種別 | ○ | 消費期限 / 賞味期限 |
  | 期限日 | ○ | 日付選択 |
  | 購入日 | - | 日付選択（デフォルト: 登録日） |
  | カテゴリ | ○ | 選択式 |
  | 保存場所 | ○ | 選択式 |
  | メモ | - | 自由入力 |

### 7.2 通知タイミング
| 期限種別 | 通知タイプ | タイミング | 内容 |
|---------|-----------|----------|------|
| 消費期限 | 期限間近 | 3日前 | 「〇〇の消費期限が3日後です」 |
| 消費期限 | 期限直前 | 1日前 | 「〇〇の消費期限が明日です！」 |
| 消費期限 | 期限当日 | 当日 | 「〇〇の消費期限が今日です！」 |
| 消費期限 | 期限切れ | 翌日 | 「⚠️ 〇〇が消費期限切れです」 |
| 賞味期限 | 期限間近 | 7日前 | 「〇〇の賞味期限が1週間後です」 |
| 賞味期限 | 期限直前 | 3日前 | 「〇〇の賞味期限が3日後です」 |
| 賞味期限 | 期限当日 | 当日 | 「〇〇の賞味期限が今日です」 |
| 賞味期限 | 期限切れ | 翌日 | 「〇〇が賞味期限切れです」 |

### 7.3 カテゴリと保存場所
- **カテゴリ**:
  - 冷蔵
  - 冷凍
  - 常温
- **保存場所**（詳細）:
  - 冷蔵庫
  - 冷凍庫
  - 野菜室
  - 食品棚
  - その他

### 7.4 単位

- パック
- 袋
- 箱
- g（グラム）
- kg（キログラム）
